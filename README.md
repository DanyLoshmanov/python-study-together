## Основные команды в GIT для Олега
- git pull                     # подтянуть свежие изменения
- git checkout -b имя-ветки    # создать и перейти в новую ветку
- git add .                    # добавить все изменения
- git commit -m "сообщение"    # закоммитить
- git push origin имя-ветки    # запушить ветку


## Основные команды в GIT для Данила
1. git pull - "Каждый раз, когда открываешь проект — подтянуть свежие изменения от друга;
2. git status - «А что вообще изменилось?» — самая часто используемая команда;
3. git checkout main  →  git pull - Перейти в главную ветку и обновить её;
4. git checkout -b new-branch-name - "Создать и сразу перейти в новую ветку (например, когда начинаешь новый проект или фикс)";
5. git add . - Добавить все изменения (или git add file.py — только один файл);
6. "git commit -m ""понятное сообщение""" - Зафиксировать изменения локально;
7. git push origin имя-ветки - Запушить свою ветку на GitHub (после этого можно делать ПР);
8. git log --oneline --graph --all - "Красиво посмотреть всю историю веток (когда запутаетесь, где что)";
9. git branch -a - Посмотреть все ветки (локальные и удалённые);
10. git merge main  (или git rebase main) - Когда закончил задачу и хочешь обновить свою ветку последними изменениями из main перед ПР;
