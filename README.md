## Основные команды в GIT для Олега
git pull                     # подтянуть свежие изменения
git checkout -b имя-ветки    # создать и перейти в новую ветку
git add .                    # добавить все изменения
git commit -m "сообщение"    # закоммитить
git push origin имя-ветки    # запушить ветку


## Основные команды в GIT для Данила
#,Команда + пример,Когда и зачем используешь ты лично,Частота
1,git pull,"Каждый раз, когда открываешь проект — подтянуть свежие изменения от друга",20 раз в день
2,git status,«А что вообще изменилось?» — самая часто используемая команда,50 раз в день
3,git checkout main  →  git pull,Перейти в главную ветку и обновить её,Каждый день
4,git checkout -b new-branch-name,"Создать и сразу перейти в новую ветку (например, когда начинаешь новый проект или фикс)",2–5 раз в неделю
5,git add .,Добавить все изменения (или git add file.py — только один файл),Каждый коммит
6,"git commit -m ""понятное сообщение""",Зафиксировать изменения локально,Каждый коммит
7,git push origin имя-ветки,Запушить свою ветку на GitHub (после этого можно делать ПР),Каждый коммит
8,git log --oneline --graph --all,"Красиво посмотреть всю историю веток (когда запутаетесь, где что)",Когда бардак
9,git branch -a,Посмотреть все ветки (локальные и удалённые),Часто
10,git merge main  (или git rebase main),Когда закончил задачу и хочешь обновить свою ветку последними изменениями из main перед ПР,Перед каждым ПР
